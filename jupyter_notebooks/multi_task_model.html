Multi Task Transfer Learning using PyTorch / FastAi¶

Multi task learning is when a model is trying to learn multiple things at the same time, for example a computer vision model trying to segment the image into different regions, finding bounding boxes of the objects detected and also classifying the objects detected in the image. All this in one go. Simultanously.

More technically...

  • Multi-task learning (MTL) is a subfield of machine learning in which multiple learning tasks are solved at the same time, while exploiting commonalities and differences across tasks.

  • This can result in improved learning efficiency and prediction accuracy for the task-specific models, when compared to training the models separately

  • In the context of Deep Learning, multi-task learning is typically done with either hard or soft parameter sharing of hidden layers.RUDER

  1. Hard Parameter Sharing

    • In hard parameter sharing we usually have a number of shared layers which learn the common patterns/features and then for each task we have a custom head attached to these shared layers
    • This type of parameter sharing has an added advantage of acting as an regularization effect. This is because now the model is trying to be good at multiple things at the same time and therefore has to generalize better.

  1. Soft Parameter Sharing

    • In soft parameter sharing on the other hand, each task has its own model with its own parameters. The distance between the parameters of the model is then regularized in order to encourage the parameters to be similar. RUDER

In this notebook we try to solve this problem.

Goal is to solve 3 different problems simultaneously:

  • classification, detection, and segmentation.

The dataset includes ~1000 images including classification, bounding box, and segmentation labels. Importantly, each image only contains one brick, ball or cylinder.

Approach¶

Our approach is going to be

  1. Load the data using FastAi
  2. Build a model using Transfer learning
  3. Build our custom heads for hard parameter sharing MLT model.

Expected Learning Outcomes¶

Learning outcome here should be

  1. How to do transfer learning
  2. Combining loss functions for multiple tasks
  3. Hyper Parameter tuning and custom head design

NOTE¶

One crucial thing to note here is that whenever we plan to use transfer learning, we have to keep in mind the assumptions made by the pretrained model that we are going to use.

For example ,for pretrained models in pytorch

All pre-trained models expect input images normalized in the some way, e.g mini-batches of 3-channel RGB images of shape (3 x H x W), where H and W are expected to be at least 224. The images have to be loaded in to a range of [0, 1] and then normalized using mean = [0.485, 0.456, 0.406] and std = [0.229, 0.224, 0.225].

FastAi provides a helper function data.normalize that takes in the as parameter the statistics like mean and std to normalize the input images. It also provides variables which have meta information stored for commonly used models like for ImageNet in imagenet_stats variable.

Therefore in the last line above we normalize the input images as needed.

We create two helper functions one to extract the class labels and one to extract the bounding box from the segmentation mask.( We later encapsulate all the helper functions in a class below, to avoid namespace pollution)

get_class_labels - relies on the fact that the mask image has only two values 0 and class label. Therefore it looks up the unique values in the image array and picks the first non-zero one.

get_bboxes - relies on first and last non-zero entries in the image to get the coordinates of the top left and bottom right points of the bounding box.

In [ ]:
def get_class_labels(self,y_true):
    return torch.tensor([torch.unique(y_true[i][y_true[i]!=0])[0] for i in range(y_true.shape[0])] , device=device)

def get_bboxes(self,y_true):
  bboxes=torch.zeros((y_true.shape[0], 4) , device=device)
  for i in range(y_true.shape[0]):
      rows,cols= torch.where(y_true[i, 0]!=0)
      bboxes[i, :] = torch.tensor([rows.min(), cols.min(), rows.max(), cols.max()])
  return bboxes / 256.

Creating the Custom Head.¶

Now we create our custom head that we will stick on top of ResNet50 model.

Our design for the custom head will have 3 seperate heads , one for each of the task at hand.

  1. Segmentation Head
    • For segmentation head we use Convolution Transpose layer which is the inverse operation of convolution and therefore increases the dimension of the input.
    • The input that this head recives is the output of the ResNet base model, which has 2048 channels
    • Then we reduce the number of channels/filters down to 4, one for each of the 4 possible segmentation outcomes. (brick , ball , cylinder , backgroun)
    • Key thing here is that we have to keep in mind that our output size is 4x256x256 and therefore we need to setup our kernel size , stride , paddings appropriately
In [ ]:
def conv_trans(self,ni, nf, ks = 4, stride = 2, padding = 1):
      return nn.Sequential(
          nn.ConvTranspose2d(ni, nf, kernel_size=ks, bias=False, stride=stride, padding = padding),
          nn.LeakyReLU(inplace = True),
          nn.BatchNorm2d(nf))

segment_model = nn.Sequential(
                            conv_trans(2048, 512,ks=7 , stride=2 , padding=1),
                            nn.Dropout(0.5),
                            conv_trans(512, 256 , ks=7 , stride=2 , padding=4),
                            conv_trans(256, 128, ks=7 , stride=2, padding=6),
                            nn.Dropout(0.5),
                            conv_trans(128, 64, ks=5 , stride=2 , padding=1),
                            conv_trans(64, 32 ,  ks=5 , stride=2 , padding=1),
                            nn.ConvTranspose2d(32, 4, kernel_size=4, bias=False, stride=1, padding = 1)
      )
  1. Bounding box head
    • For bounding box head we Flatten the output of the ResNet and then pass it through a bunch of Linear layers
    • The number of output units is 4, (x-top-left , y-top-left , x-bottom-right , y-bottom-right)

  1. Classification head
    • For classification head, we follow a standard classifier design
    • Similar to the bounding box model , we Flatten the ResNet CNN output and then pass it through linear layers

Forward method of our MLT¶

moving the data forward through our network we take the output from resnet and pass it separately to each of the 3 custom heads we have.

In [ ]:
def forward(x):
      segment = segment_model(x)
      #print(x.shape)
      bounding_box = bounding_box_model(x)
      classification = classification_model(x)
      return bounding_box , segment , classification

Important bit is that we would be returning 3 different outputs and then use each of those in our custom loss function to calculate how good or bad we are doing.

Custom Loss function¶

Our loss function we will be calculating individual losses for each of our tasks.

  1. For classification its straightforward that we use Cross Entropy
  2. For regression/bounding box, we can use MSE or L1 loss
  3. For segmentation, there are many segmentation specific loss functions. A good survey is here. But since we are outputing 4 channels in our segment model, one for each of the class, we can also use Cross Entropy as well.

BUT how do we combine all these individual losses ?

Now this is the challenging bit, one option is to introduce more hyperparameters that weight each loss and then we simply combine/add them together to get a total loss.

There are other more sophisticated approaches but lets go for simple weighted combination.

def my_loss_function(y_pred,y_true):
    class_labels = get_class_labels(y_true)
    #print(class_labels)
    bboxes =  get_bboxes(y_true)

    mseloss = nn.SmoothL1Loss()
    cel1 = nn.CrossEntropyLoss(weight=torch.tensor([0.1,1,1,1] , device=device))
    cel2 = nn.CrossEntropyLoss(weight=torch.tensor([0.3,1,1,1] , device=device))

    class_loss  =cel1(y_pred[2] , class_labels)
    bb_loss = mseloss(y_pred[0] , bboxes )
    segment_loss = cel2( y_pred[1] , y_true.squeeze(1) )


    #class_loss = 0
    #bb_loss = 0
    total_loss = torch.div(5*segment_loss + 5*class_loss + 15*bb_loss , 25)
    print(f"class_loss : {class_loss} , bb_loss: {bb_loss} , segment_loss : {segment_loss} ,  total loss {total_loss}")
    return total_loss

      classification = self.classification_model(x)
      return bounding_box , segment , classification

    return torch.tensor([torch.unique(y_true[i][y_true[i]!=0])[0] for i in range(y_true.shape[0])] , device=device)

  def my_loss_function(self,y_pred,y_true):
    class_labels = self.get_class_labels(y_true)
    #print(class_labels)
    bboxes =  self.get_bboxes(y_true)

    mseloss = nn.SmoothL1Loss()
    cel1 = nn.CrossEntropyLoss(weight=torch.tensor([0.1,1,1,1] , device=device))
    cel2 = nn.CrossEntropyLoss(weight=torch.tensor([0.3,1,1,1] , device=device))

    class_loss  =cel1(y_pred[2] , class_labels)
    bb_loss = mseloss(y_pred[0] , bboxes )
    segment_loss = cel2( y_pred[1] , y_true.squeeze(1) )


    #class_loss = 0
    #bb_loss = 0
    total_loss = torch.div(5*segment_loss + 5*class_loss + 15*bb_loss , 25)
    print(f"class_loss : {class_loss} , bb_loss: {bb_loss} , segment_loss : {segment_loss} ,  total loss {total_loss}")
    return total_loss

  def pixel_accuracy(self,y_pred, y_true):
    y_=y_true.squeeze(dim=1)
    yhat_=y_pred[1].argmax(dim=1)
    return (y_==yhat_).sum().float()/y_true.numel()

  def pixel_recall(self, y_pred , y_true):
    y_=y_true.squeeze(dim=1)
    yhat_=y_pred[1].argmax(dim=1)
    return (y_[yhat_ > 0]==yhat_[yhat_ > 0]).sum().float()/y_[y_ > 0].numel()

from fastai.vision import load_learner, normalize, torch
import numpy as np

class Model(object):
    def __init__(self, path='../sample_models', file='export.pkl'):

        self.learn=load_learner(path=path, file=file) #Load model
        self.class_names=['brick', 'ball', 'cylinder'] #Be careful here, labeled data uses this order, but fastai will use alphabetical by default!

    def predict(self, x):
        '''
        Input: x = block of input images, stored as Torch.Tensor of dimension (batch_sizex3xHxW), 
                   scaled between 0 and 1. 
        Returns: a tuple containing: 
            1. The final class predictions for each image (brick, ball, or cylinder) as a list of strings.
            2. Upper left and lower right bounding box coordinates (in pixels) for the brick ball 
            or cylinder in each image, as a 2d numpy array of dimension batch_size x 4.
            3. Segmentation mask for the image, as a 3d numpy array of dimension (batch_sizexHxW). Each value 
            in each segmentation mask should be either 0, 1, 2, or 3. Where 0=background, 1=brick, 
            2=ball, 3=cylinder. 
        '''

        #Normalize input data using the same mean and std used in training:
        x_norm=normalize(x, torch.tensor(self.learn.data.stats[0]),
                            torch.tensor(self.learn.data.stats[1]))


        #Pass data into model:
        device = torch.device('cuda' if torch.cuda.is_available() else 'cpu')
        with torch.no_grad():
            bounding , segment , classification =self.learn.model(x_norm.to(device))
            #print(bounding , segment)
            bounding = bounding.detach().cpu().numpy()
            segment = segment.detach().cpu().numpy()
            classification = classification.detach().cpu().numpy()

        #Post-processing/parsing outputs, here's an example for classification only:
        class_prediction_indices= np.argmax(classification , 1)
        class_predictions=[self.learn.data.classes[i] for i in class_prediction_indices]

        return (class_predictions, bounding*256., np.argmax(segment, 1))

import numpy as np
import time
from fastai.vision import *
from pathlib import Path

def evaluate(data_path='../data/bbc_train',
             model_dir = '../sample_models',
             batch_size=64,
             im_size=(256,256)):


    print("1. Loading Data...")

    path=Path(data_path)
    def get_y_fn(x): return path/'masks'/(x.stem + '.png')
    classes = array(['background', 'brick', 'ball', 'cylinder'])

    src = (SegmentationItemList.from_folder(path/'images')
       .split_by_rand_pct(0.0)
       .label_from_func(get_y_fn, classes=classes))

    #Don't normalize data here - assume normalization happens inside of Model. 
    data = src.transform(get_transforms(), tfm_y=True, size=im_size).databunch(bs=batch_size)
    print(data)


    print("2. Instantiating Model...")
    M = Model(path=model_dir, file='export2.pkl')


    print("3. Evaluating on Test Images...")
    round_metric = []
    for i in range(5):
      x,y = data.one_batch()
      #Extract class label from mask:
      class_labels=np.array([np.unique(y[i][y[i]!=0])[0] for i in range(x.shape[0])])

      #Extract bounding box from mask:
      bboxes=np.zeros((x.shape[0], 4))
      for i in range(x.shape[0]):
          rows,cols= np.where(y[i, 0]!=0)
          bboxes[i, :] = np.array([rows.min(), cols.min(), rows.max(), cols.max()])

      preds=M.predict(x)

      classification_accuracy=(np.array([data.classes[i] for i in class_labels])==np.array(preds[0])).sum()/len(preds[0])
      bbox_score = 1 - np.mean(np.abs(bboxes-preds[1]))/(x.shape[-1]) #Divide by image height to rougly normalize score
      segmentation_accuracy=float((preds[2] == np.array(y.squeeze(1))).sum())/y.numel()
      combined_accuracy=np.mean([classification_accuracy, bbox_score, segmentation_accuracy])
      round_metric.append([combined_accuracy, classification_accuracy, bbox_score, segmentation_accuracy])

    return np.min(round_metric , 1)


def calculate_score(combined_accuracy):
    if combined_accuracy >= 0.99: score = 10
    elif combined_accuracy >= 0.98: score = 9
    elif combined_accuracy >= 0.97: score = 8
    elif combined_accuracy >= 0.96: score = 7
    elif combined_accuracy >= 0.95: score = 6
    elif combined_accuracy >= 0.90: score = 5
    else: score = 4
    return score
